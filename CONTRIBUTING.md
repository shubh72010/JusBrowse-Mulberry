# Contributing

## Development Workflow

1. Fork the repository.
2. Follow the [Installation Guide](INSTALLATION.md) to build.
3. Create a branch for your change.
4. Open a pull request against `main`.

## Security Disclosures

Report critical vulnerabilities privately via GitHub's security advisory tool. Do not open a public issue.
